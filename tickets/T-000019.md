## Goal
Introduce a centralized theme token system for app surfaces, typography, and controls.

## Acceptance Criteria
- [x] Core UI colors (page, text, panels, controls, modal, board surfaces) are defined via centralized CSS custom properties.
- [x] Existing core styles use semantic theme tokens instead of direct hard-coded hex values.
- [x] A reusable theme foundation helper exists in app code to set active theme id and apply/clear token overrides.
- [x] Default theme is applied at app startup and baseline render (`data-theme="default"`).
- [x] Project build passes after theme foundation refactor.

## Goal
Add a theme selector in UI, apply default theme on first load, and persist user choice locally.

## Acceptance Criteria
- [x] A visible Theme selector is present in the main UI and lists all built-in themes.
- [x] Theme selection applies immediately at runtime by switching active theme id on the document.
- [x] Selected theme is persisted to local storage and restored on next page load.
- [x] Invalid/missing stored theme falls back safely to default theme.
- [x] Project build passes with theme switcher wiring.

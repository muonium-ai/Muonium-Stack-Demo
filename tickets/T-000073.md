## Goal
Now that the repo is public, set up GitHub Pages to host the built demo so visitors can try it without cloning.

## Acceptance Criteria
- [x] GitHub Actions workflow deploys the Vite build to GitHub Pages on push to `main`.
- [x] `vite.config.js` sets the correct `base` path for the repository.
- [x] GitHub Pages is enabled on the repo (source: GitHub Actions).
- [ ] The site is accessible at `https://muonium-ai.github.io/Muonium-Stack-Demo/`.

## Notes
- The build requires Rust/wasm-pack for the WASM chess engine.
- COOP/COEP headers are needed for SharedArrayBuffer (Stockfish threading), but GitHub Pages doesn't support custom headers — Stockfish may fall back to single-threaded mode.
- The playground (simulations/playground) is a separate Vite build and can be added later.

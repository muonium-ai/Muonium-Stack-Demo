## Goal
Build the in-browser telemetry store/stream layer (Redis-like semantics) for physics metrics.

## Acceptance Criteria
- [x] Frame-level events are sampled and aggregated at configurable intervals.
- [x] Hash/list-style metric APIs support counters, gauges, and timeline events.
- [x] Streaming subscription API feeds HUD and graph consumers in real time.
- [x] Metrics pipeline sustains target throughput without blocking simulation loop.

## Goal
Add top-level tabs so users can switch between `Basic` and `Advanced` experiences in the same isolated playground app.

## Acceptance Criteria
- [x] Tab navigation exists with exactly two modes: `Basic` and `Advanced`.
- [x] Mode switch does not break simulation correctness or runtime state handling.
- [x] Basic mode presents simplified controls; Advanced mode preserves full controls.
- [x] URL/state persistence for active tab is supported (query param or local storage).

## Notes
- Current UI should become the baseline for Advanced mode.
